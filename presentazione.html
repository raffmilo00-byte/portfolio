02

Tecnologie utilizzate

  • HTML — struttura semantica, sezioni, form accessibile, meta tag Open Graph.
  • CSS — layout responsive (Grid/Flexbox), variabili, tema scuro, tipografia fluida.
  • JavaScript (minimo) — anno nel footer; form contatti (EmailJS opzionale o mailto).

Nessun framework: dimostrazione delle competenze di base richieste dal modulo.

03

Obiettivo e pubblico

Il portfolio presenta il mio profilo a aziende che formano junior web developer, valorizzando anche l’esperienza in assistenza clienti e reti Wi-Fi (Wirlab, dal 10 dicembre 2024). Il messaggio è chiaro: studio con Start2Impact, lavoro in ambito tech/supporto e punto a crescere verso l’AI Engineering.

Struttura one-page con navigazione ancorata: hero → chi sono → cosa cerco → competenze → progetti → contatti. CV separato in cv.html per chi vuole il formato stampabile.

04

Scelte di design e contenuto

Tema scuro e accenti ciano/viola per un look tech moderno; sezioni alternate per scandire la lettura. Foto professionale in hero per umanizzare il profilo; card Wirlab per credibilità lavorativa.

Competenze a “chip” (HTML, CSS, JS, reti Wi-Fi, inglese buono). Progetti Demetra e GreenEarth in cartella progetti/ per mostrare progressione. Form contatti senza email in chiaro (anti-spam) e checkbox privacy.

05

Accessibilità e pubblicazione

Skip-link, etichette ARIA, focus visibile sui campi, lang="it", immagini con alt. Repository ordinato: solo il portfolio alla radice del repo (non cartelle cv/pagina web separate).

Pubblicazione tramite GitHub Pages (branch main, cartella root). File .nojekyll per servire correttamente i file statici.

Visita il sito: raffmilo00-byte.github.io/il-mio-porfolio/PORTFOLIO.html
